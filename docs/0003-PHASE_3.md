# Phase 3: Core Pages

**Status:** Not Started
**Depends on:** Phase 2 (Data Pipeline) — complete
**Ref:** See `0000-PLAN.md` §Implementation Order, items 13–17

---

## Goal

Build the five core pages of the application: Dashboard, Fixtures, League Table, Match Detail, and Odds Board. All pages are server-rendered Django templates using HTMX for partial updates. No JavaScript frameworks — interactivity comes from HTMX attributes and Django template partials.

---

## Tasks

### 3.1 — Template Partials & Shared Components
**Files:** `matches/templates/matches/partials/`, `betting/templates/betting/partials/`

Partials are small template fragments returned by HTMX requests. They render independently (no `{% extends %}`) and are swapped into the page.

- [ ] `matches/partials/match_card.html` — Single match card (teams, score, status badge, kickoff time, top odds). Reused on Dashboard, Fixtures, and Match Detail.
- [ ] `matches/partials/standings_row.html` — Single table row for a standing entry (position, team crest+name, P/W/D/L/GF/GA/GD/Pts). Reused on League Table.
- [ ] `betting/partials/odds_row.html` — Single row for an odds comparison (match info + bookmaker odds columns). Reused on Odds Board and Match Detail.
- [ ] `website/templates/website/components/matchday_tabs.html` — Horizontal tab bar for selecting matchday (1–38). Active tab highlighted.

### 3.2 — Dashboard (`/`)
**Files:** `matches/views.py`, `matches/templates/matches/dashboard.html`

The home page. Shows today's (or the nearest matchday's) fixtures as a grid of match cards.

- [ ] `DashboardView` — TemplateView or function view
  - Query: `Match` objects for today, or fall back to the most recent/next matchday
  - Context: `matches`, `current_matchday`
  - For each match, prefetch the best odds (lowest home/draw/away from any bookmaker)
- [ ] `dashboard.html` — Extends `base.html`
  - Responsive grid of `match_card.html` partials (2-col on desktop, 1-col mobile)
  - Status-based styling: green glow for LIVE matches, muted for FINISHED
  - "No matches today" empty state
  - Each card links to Match Detail (`/match/<id>/`)
- [ ] HTMX: Cards are wrapped in `hx-get` containers for future WebSocket swap (Phase 4)
- [ ] Update `website/urls.py` or `config/urls.py` to route `/` to `DashboardView`

### 3.3 — Fixtures (`/fixtures/`)
**Files:** `matches/views.py`, `matches/templates/matches/fixtures.html`, `matches/templates/matches/partials/fixture_list.html`

Browse all matches organized by matchday. Tab interface to switch between matchdays.

- [ ] `FixturesView` — Accepts `?matchday=N` query param (default: current matchday)
  - Query: `Match.objects.filter(season=CURRENT_SEASON, matchday=matchday)` with `select_related('home_team', 'away_team')`
  - Context: `matches`, `matchday`, `matchdays` (range 1–38), `current_matchday`
- [ ] `fixtures.html` — Extends `base.html`
  - Matchday tab bar across the top (horizontally scrollable on mobile)
  - Match list grouped by date within the matchday
  - Score + status for each match, or "vs" if not started
- [ ] `partials/fixture_list.html` — Just the match list (no layout wrapper). Returned by HTMX tab clicks.
- [ ] HTMX: Matchday tabs use `hx-get="/fixtures/?matchday=N"` with `hx-target="#fixture-list"` and `hx-swap="innerHTML"` to swap just the match list, plus `hx-push-url="true"` for URL updates
- [ ] URL: `/fixtures/` in `matches/urls.py`

### 3.4 — League Table (`/table/`)
**Files:** `matches/views.py`, `matches/templates/matches/league_table.html`

Full EPL standings. Clean data table with team crests.

- [ ] `LeagueTableView` — Fetches `Standing.objects.filter(season=CURRENT_SEASON).select_related('team').order_by('position')`
  - Context: `standings`, `season`
- [ ] `league_table.html` — Extends `base.html`
  - Full-width responsive table: Pos / Team (crest + name) / P / W / D / L / GF / GA / GD / Pts
  - Top 4 highlighted (Champions League), bottom 3 highlighted (relegation)
  - Team names link to a filtered fixtures view (`/fixtures/?team=<id>`) — optional, nice-to-have
- [ ] HTMX: Table body wrapped in an `hx-get` target for future live refresh (Phase 4)
- [ ] URL: `/table/` in `matches/urls.py`

### 3.5 — Match Detail (`/match/<id>/`)
**Files:** `matches/views.py`, `matches/templates/matches/match_detail.html`

Detailed view of a single match with odds comparison and (later) bet form.

- [ ] `MatchDetailView` — `DetailView` on `Match`, prefetch `odds_set` and related teams
  - Context: `match`, `odds` (grouped by bookmaker), `best_odds` (lowest for each outcome)
- [ ] `match_detail.html` — Extends `base.html`
  - Header: Home team (crest + name) vs Away team, score (if started), status badge, kickoff datetime, matchday, venue
  - Odds comparison table: rows = bookmakers, columns = Home / Draw / Away, best odds highlighted in green
  - If no odds: "Odds not yet available" placeholder
  - Placeholder section for bet form (Phase 5)
- [ ] HTMX: Odds table has `id="odds-table"` for future auto-refresh (`hx-trigger="every 30s"` in Phase 4)
- [ ] URL: `/match/<int:pk>/` in `matches/urls.py`

### 3.6 — Odds Board (`/odds/`)
**Files:** `betting/views.py`, `betting/templates/betting/odds_board.html`

All upcoming matches with their best odds from any bookmaker. Quick overview for bettors.

- [ ] `OddsBoardView` — Fetches upcoming matches (status in `SCHEDULED`, `TIMED`) with prefetched odds
  - Context: `matches_with_odds` — list of dicts with match info + best home/draw/away odds across all bookmakers
  - Ordered by kickoff time
- [ ] `odds_board.html` — Extends `base.html`
  - Table or card layout: Kickoff / Home Team / Best Home Odds / Best Draw Odds / Best Away Odds / Away Team
  - Best odds highlighted
  - Each row links to Match Detail
  - "No upcoming matches with odds" empty state
- [ ] HTMX: Table has `id="odds-board"` for future auto-refresh (Phase 4)
- [ ] URL: `/odds/` in `betting/urls.py`

### 3.7 — URL Wiring & Navigation
**Files:** `matches/urls.py`, `betting/urls.py`, `config/urls.py`, navbar component

- [ ] Register all new URL patterns:
  - `/` → `DashboardView` (update from current `HomeView`)
  - `/fixtures/` → `FixturesView`
  - `/table/` → `LeagueTableView`
  - `/match/<int:pk>/` → `MatchDetailView`
  - `/odds/` → `OddsBoardView`
- [ ] Update `navbar.html` links to point to the actual URL names
- [ ] Add `{% url %}` tags throughout templates for URL resolution
- [ ] Set active state on navbar for current page

### 3.8 — Custom Template Tags
**File:** `matches/templatetags/match_tags.py`

Reusable display helpers to keep templates clean.

- [ ] `{% status_badge match %}` — Renders a colored badge based on match status (green for LIVE, gray for SCHEDULED, etc.)
- [ ] `{% score_display match %}` — Shows "2 - 1" if finished/live, or "vs" if not started
- [ ] `{{ odds_value|format_odds }}` — Formats decimal odds to 2 decimal places
- [ ] Register the templatetags module (`matches/templatetags/__init__.py`)

## Design Notes

### Match Card Layout
```
┌──────────────────────────────┐
│  ● LIVE 63'          MD 28   │  ← status badge + matchday
│                              │
│  Arsenal        2            │  ← home team + score
│  Chelsea        1            │  ← away team + score
│                              │
│  1.45  4.20  6.50           │  ← best H/D/A odds
│  SAT 15:00                  │  ← or kickoff if not started
└──────────────────────────────┘
```

### Standings Table Row
```
Pos  Team                    P   W   D   L   GF  GA  GD  Pts
 1   🛡 Arsenal              28  20  5   3   62  24  +38  65
```

### Status Badge Colors
| Status | Color | Label |
|--------|-------|-------|
| `SCHEDULED` / `TIMED` | Gray (`text-gray-400`) | Kickoff time |
| `IN_PLAY` / `PAUSED` | Green (`text-accent`) | "LIVE" + minute |
| `FINISHED` | Muted (`text-gray-500`) | "FT" |
| `POSTPONED` | Amber (`text-warning`) | "PP" |
| `CANCELLED` | Red (`text-danger`) | "CAN" |

## Suggested Implementation Order

1. **3.8** — Template tags (small, reused by everything)
2. **3.1** — Partials (match card, standings row, odds row)
3. **3.4** — League Table (simplest full page — single query, one table)
4. **3.2** — Dashboard (uses match cards, moderate query logic)
5. **3.3** — Fixtures (adds HTMX tab interaction)
6. **3.5** — Match Detail (odds table, most complex layout)
7. **3.6** — Odds Board (depends on odds data + match detail patterns)
8. **3.7** — URL wiring & nav updates (tie it all together)

This order builds from simple to complex, with each page reusing partials from earlier steps.

- [ ] Dashboard shows matches for the current/nearest matchday with scores and status badges
- [ ] Fixtures page loads different matchdays via HTMX tab clicks without full page reload
- [ ] League Table displays all 20 teams in correct order with CL/relegation zone highlighting
- [ ] Match Detail shows odds comparison table with best odds highlighted
- [ ] Odds Board lists all upcoming matches with their best available odds
- [ ] All pages are responsive (usable at 375px mobile width)
