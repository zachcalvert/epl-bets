# Phase 9: Deploy & Publish

## Status: Complete

**Depends on:** Phase 8 (CI/CD) — complete
**Ref:** See `LAUNCH-PLAN.md` §Phase 9

## Goal

Ship the project publicly with a working production deployment, a shareable live URL, and an open-source repo that reflects the real state of the app.

## Implemented Slice

### 9A — Fly Deployment

Phase 9 now has a live Fly deployment serving the app at `https://eplbets.net`.

Production deployment includes:
- `fly.toml` checked into the repo
- Fly app/runtime configuration for the Django web process
- production domain wired up and reachable
- deploy automation from GitHub Actions using `FLY_API_TOKEN`
- Neon-backed PostgreSQL persistence verified from the live app after production seeding

### 9B — Public README and Demo Link

`README.md` now reflects the production state of the project with:
- live demo link near the top of the document
- deployment section calling out Fly hosting
- GitHub Actions and Codecov badges for public repo visibility

### 9C — Public Repository

The project is now published at `https://github.com/zachcalvert/epl-bets`.

That gives the portfolio demo:
- public source visibility
- CI status surfaced on the repo landing page
- a deploy path connected to pushes on `main`

## Outcome

Phase 9 is complete because the app is no longer just locally runnable; it is publicly reachable, documented as such, and packaged as an open-source portfolio project with automated deployment support.

## Follow-On Gaps

Phase 9 intentionally stops short of deeper production operations work. The next planning pass should focus on:
- observability and error reporting
- deploy smoke checks and operational runbooks
- Fly process-group sizing and memory guidance for `app`, `worker`, and `beat`
- performance/caching review under production traffic
- richer portfolio-facing transparency features
