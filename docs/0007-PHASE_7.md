## Overview

Phase 7 is partially complete. The repo now has the core community files, a reusable pytest scaffold, and an initial automated test suite for the most self-contained app surfaces. That establishes the testing baseline, but the original Phase 7 goal of broad source coverage is still unfinished because `matches` and `betting` remain largely untested.

### 7B — Test Scaffold

**New dev dependencies** (added to `pyproject.toml`):
- `pytest` — test runner
- `pytest-django` — Django integration (settings, db fixtures, client)
- `pytest-cov` — coverage reporting with `fail_under` gate
- `pytest-xdist` — parallel test execution
- `factory-boy` — model factories with `SubFactory` for FK relationships
- `respx` — httpx mock library (purpose-built for the project's HTTP client)

**pytest + coverage config** (added to `pyproject.toml`):
- `[tool.pytest.ini_options]` — `DJANGO_SETTINGS_MODULE`, strict markers, verbose output
- `[tool.coverage.run]` — source apps, omit migrations/tests/admin
- `[tool.coverage.report]` — `fail_under = 98`

**New files:**
- `codecov.yml` — project target 90%, patch target 80%
- `conftest.py` (root) — autouse settings fixture overriding channel layer to `InMemoryChannelLayer`, setting `CELERY_TASK_ALWAYS_EAGER`, and providing test API keys

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| HTTP mocking | `respx` | Purpose-built for httpx; cleaner than patching with unittest.mock |
| Fixture strategy | factory-boy per app | SubFactory handles FK wiring; DRY across test files |
| HTMX testing | `HTTP_HX_REQUEST` header + template assertions | django-htmx middleware checks this header |
| Test organization | `<app>/tests/test_<module>.py` | Clear mapping to source; pytest-xdist parallelizes at file level |
