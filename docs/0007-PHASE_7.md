# Phase 7: Community & Testing

## Status: In Progress

**Depends on:** Phase 6 (Polish) — complete
**Ref:** See `LAUNCH-PLAN.md` §Phase 7

## Overview

Phase 7 prepares the codebase for public release by adding community files and a comprehensive pytest test suite targeting 90%+ coverage. No existing tests — this builds the entire test infrastructure from scratch.

### 7B — Test Scaffold

**New dev dependencies** (added to `pyproject.toml`):
- `pytest` — test runner
- `pytest-django` — Django integration (settings, db fixtures, client)
- `pytest-cov` — coverage reporting with `fail_under` gate
- `pytest-xdist` — parallel test execution
- `factory-boy` — model factories with `SubFactory` for FK relationships
- `respx` — httpx mock library (purpose-built for the project's HTTP client)

**pytest + coverage config** (added to `pyproject.toml`):
- `[tool.pytest.ini_options]` — `DJANGO_SETTINGS_MODULE`, strict markers, verbose output
- `[tool.coverage.run]` — source apps, omit migrations/tests/admin
- `[tool.coverage.report]` — `fail_under = 90`

**New files:**
- `codecov.yml` — project target 90%, patch target 80%
- `conftest.py` (root) — autouse settings fixture overriding channel layer to `InMemoryChannelLayer`, setting `CELERY_TASK_ALWAYS_EAGER`, and providing test API keys

### 7D — Tests: matches app (~54 tests)

**Models (~10):** Team/Match/Standing CRUD, FK relationships, unique constraints, ordering, `__str__`

**Services (~12):** `FootballDataClient` methods mocked with `respx`. Sync helpers tested for create/update counts, missing team handling, rate limit errors.

**Tasks (~6):** Each `fetch_*` task tested by patching its sync helper. `fetch_live_scores` additionally verifies broadcast logic via patched channel layer.

**Views (~14):** Status codes, template names, context data, HTMX partial responses (via `HTTP_HX_REQUEST` header), matchday query params, authenticated vs anonymous contexts.

**Consumers (~5):** `WebsocketCommunicator` tests for connect/disconnect, dashboard and match-scoped groups, receiving `score_update` and `match_score_update` events.

**Template tags (~7):** `status_badge` for each match status, `score_display` with/without scores, `format_odds` for decimals and `None`.

### 7E — Tests: betting app (~46 tests)

**Models (~8):** Odds/BetSlip/UserBalance CRUD, unique_together constraints, status/selection choices, default balance.

**Services (~8):** `OddsApiClient` mocked with `respx`, credit tracking, `sync_odds` with team name resolution (direct match + alias), unresolved team handling.

**Tasks (~10):** `fetch_odds` with patched sync. `settle_match_bets` is the most complex — tested for home win, away win, draw, losing bet, void (cancelled), void (postponed), no pending bets, and missing match.

**Views (~14):** OddsBoard, PlaceBet (auth, valid submission, insufficient balance, match not accepting, no odds, invalid form), MyBets (auth, with bets, empty), QuickBetForm (auth, returns form). HTMX partials.

**Forms (~6):** PlaceBetForm validation — valid, stake too low/high, missing selection, invalid selection, decimal precision.

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| HTTP mocking | `respx` | Purpose-built for httpx; cleaner than patching with unittest.mock |
| Fixture strategy | factory-boy per app | SubFactory handles FK wiring; DRY across test files |
| Celery testing | Call tasks directly, patch sync helpers | No broker needed; tasks are thin wrappers |
| WebSocket testing | `channels.testing.WebsocketCommunicator` | Ships with Channels; tests full lifecycle |
| HTMX testing | `HTTP_HX_REQUEST` header + template assertions | django-htmx middleware checks this header |
| Test organization | `<app>/tests/test_<module>.py` | Clear mapping to source; pytest-xdist parallelizes at file level |
