## Summary

Phase 2 built the full data ingestion layer: API clients for football-data.org and The Odds API, sync helpers, Celery tasks with Beat scheduling, a management command for seeding, and static JSON for offline development.

---

## What Was Built

### API Clients

**`matches/services.py` ŌĆö `FootballDataClient`**
- httpx-based client wrapping football-data.org v4 API
- Methods: `get_teams(season)`, `get_matches(season, matchday, status)`, `get_match(match_id)`, `get_standings(season)`
- Auth via `X-Auth-Token` header, rate limit detection (raises `RateLimitError` on 429)
- Response normalization: flattens nested JSON, parses ISO datetimes, maps to model-friendly dicts
- Context manager support for clean resource handling

**`betting/services.py` ŌĆö `OddsApiClient`**
- httpx-based client wrapping The Odds API v4
- Method: `get_epl_odds(markets, regions)` for h2h (1X2) odds
- Credit tracking via `x-requests-remaining` / `x-requests-used` response headers, logged after each call
- Team name fuzzy matching via normalized name lookup + alias table covering all EPL naming variations

### Sync Helpers

| Function | File | Key Behavior |
|----------|------|-------------|
| `sync_teams(season, offline)` | `matches/services.py` | `update_or_create` on `external_id`, returns `(created, updated)` |
| `sync_matches(season, matchday, status, offline)` | `matches/services.py` | Resolves team FKs by `external_id`, `update_or_create` on match `external_id` |
| `sync_standings(season, offline)` | `matches/services.py` | `update_or_create` on `(team, season)` |
| `sync_odds()` | `betting/services.py` | Matches Odds API events to `Match` records via team name lookup, `update_or_create` on `(match, bookmaker)` |

All sync functions are idempotent and log created/updated counts.

### Celery Tasks

**`matches/tasks.py`:**
| Task | Schedule | What It Does |
|------|----------|-------------|
| `fetch_teams` | Daily at 3 AM | `sync_teams(CURRENT_SEASON)` |
| `fetch_fixtures` | Every 6 hours | `sync_matches(CURRENT_SEASON)` |
| `fetch_standings` | Every 6 hours | `sync_standings(CURRENT_SEASON)` |
| `fetch_live_scores` | Every 60 seconds | `sync_matches(CURRENT_SEASON, status="LIVE")` |

**`betting/tasks.py`:**
| Task | Schedule | What It Does |
|------|----------|-------------|
| `fetch_odds` | Every 30 minutes | `sync_odds()` |

All tasks: `@shared_task(bind=True)`, max 3 retries with exponential backoff.

### Beat Schedule

`CELERY_BEAT_SCHEDULE` registered in `config/settings.py` using `crontab` and `timedelta` schedules.

### Management Command

`python manage.py seed_epl` ŌĆö sequential sync: teams ŌåÆ matches ŌåÆ standings ŌåÆ odds

| Flag | Effect |
|------|--------|
| `--season` | Season start year (default: `2025`) |
| `--skip-odds` | Skip odds fetching (saves API credits) |
| `--offline` | Seed from bundled static JSON instead of live APIs |
