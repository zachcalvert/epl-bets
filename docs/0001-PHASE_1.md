# Phase 1: Foundation

**Status:** Complete
**Commits:** `8f12a7f` through `e94317f`

---

## Summary

Phase 1 established the full project scaffold: Django apps, data models, Docker infrastructure, real-time stack configuration, admin interface, and the base frontend template with dark theme styling.

---

## What Was Built

### Project Scaffold

- Django 5.1 project under `config/` (settings, URLs, ASGI, WSGI, Celery)
- Four Django apps created: `core`, `users`, `matches`, `betting`, `website`
- `pyproject.toml` with all dependencies (Django, DRF, Channels, Celery, Redis, HTMX, httpx, etc.)
- `Dockerfile` and `entrypoint.sh` for containerized deployment
- `.env.example` with all required environment variables

### Data Models

**`core/models.py`** — Abstract `BaseModel` with auto-generated `id_hash`, `created_at`, `updated_at`

**`users/models.py`** — Custom `User` model with email-based authentication and `UserManager`

**`matches/models.py`:**
| Model | Key Fields |
|-------|-----------|
| `Team` | `external_id`, `name`, `short_name`, `tla`, `crest_url`, `venue` |
| `Match` | `home_team`/`away_team` (FK), `home_score`/`away_score`, `status` (7 choices), `matchday`, `kickoff`, `season` |
| `Standing` | `team` (FK), `season`, full stats (P/W/D/L/GF/GA/GD/Pts), unique on `(team, season)` |

**`betting/models.py`:**
| Model | Key Fields |
|-------|-----------|
| `Odds` | `match` (FK), `bookmaker`, `home_win`/`draw`/`away_win` (decimal), `fetched_at`, unique on `(match, bookmaker)` |
| `BetSlip` | `user`/`match` (FK), `selection`, `odds_at_placement`, `stake`, `status` (4 choices), `payout` |
| `UserBalance` | `user` (OneToOne), `balance` (default 1000.00) |

### Docker Compose Services

| Service | Image | Purpose |
|---------|-------|---------|
| `db` | PostgreSQL 16 | Persistent data store |
| `redis` | Redis 7-alpine | Cache, Celery broker, Channels layer |
| `web` | Custom (Dockerfile) | Django dev server on port 8000 |
| `worker` | Custom (Dockerfile) | Celery worker for background tasks |
| `beat` | Custom (Dockerfile) | Celery Beat periodic scheduler |

### Configuration

- **Celery** — `config/celery.py` with autodiscover, Redis broker/backend, `DatabaseScheduler`
- **Channels** — ASGI app in `config/asgi.py` with `ProtocolTypeRouter` (HTTP configured, WebSocket ready)
- **Redis** — Configured as cache backend, Celery broker, and Channels layer
- **JWT** — SimpleJWT configured for REST API auth
- **CORS** — Enabled in debug mode
- **Middleware** — Includes `django-htmx` middleware

### Admin Interface

All models registered with search fields and list filters:
- `Team` — searchable by name/short_name/TLA
- `Match` — filterable by status/season/matchday
- `Standing` — filterable by season
- `Odds` — filterable by bookmaker
- `BetSlip` — filterable by status/selection
- `UserBalance` — searchable by user email

### Frontend / Templates

- **`base.html`** — Master layout with Tailwind CSS (CDN), HTMX 2.0.4, htmx-ws extension, Google Fonts (Inter + Oswald)
- **`navbar.html`** — Responsive nav with desktop/mobile menus linking all planned pages
- **`footer.html`** — Demo disclaimer and tech stack info
- **`home.html`** — Landing page with hero section and feature cards
- **Dark theme** — Custom Tailwind palette: `dark` (#1a1a2e), `surface` (#16213e), `accent` (#00e676), `danger` (#ff1744), `warning` (#ffab00), `epl` (#38003c)

### URL Routing

- `config/urls.py` — Main dispatcher includes `matches/`, `betting/`, `api/`, and website root
- `website/urls.py` — Home page via `HomeView`
- `matches/urls.py` and `betting/urls.py` — Registered but empty (views not yet implemented)
