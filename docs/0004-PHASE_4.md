# Phase 4: Real-Time Features

## Status: Complete ✅

## Overview

Phase 4 adds real-time updates to the EPL Bets application using two complementary mechanisms:

1. **WebSocket** (Django Channels + htmx-ext-ws) — for live score pushes on Dashboard & Match Detail
2. **HTMX polling** (hx-trigger="every 30s") — for odds refresh on Odds Board & Match Detail

## What Was Built

### WebSocket Infrastructure
- **`matches/consumers.py`** — `LiveUpdatesConsumer` handles WebSocket connections with group-based routing:
  - `"dashboard"` scope → joins `live_scores` group for all live match updates
  - `"<match_pk>"` scope → joins `match_<pk>` group for single match updates
- **`matches/routing.py`** — WebSocket URL pattern: `ws/live/<scope>/`
- **`config/asgi.py`** — Updated to route both HTTP and WebSocket protocols via `AuthMiddlewareStack`

### OOB Swap Partials
- **`match_card_oob.html`** — Match card with `hx-swap-oob="true"` for dashboard live updates
- **`score_display_oob.html`** — Score display + status badge with OOB swap for match detail page
- Match cards now have `id="match-{{ match.pk }}"` for targeted swaps

### Celery Broadcasting
- **`matches/tasks.py`** — `fetch_live_scores()` now:
  1. Snapshots live match states before sync
  2. Detects score changes and status transitions after sync
  3. Broadcasts updates via `channel_layer.group_send()` to relevant groups
  - Consumer renders HTML from templates; Celery only sends match PKs (clean separation)

### HTMX Polling for Odds
- **Odds Board** (`/odds/`) — polls `/odds/partial/` every 30 seconds
- **Match Detail** (`/match/<pk>/`) — polls `/match/<pk>/odds/` every 30 seconds
- New partial views: `OddsBoardPartialView`, `MatchOddsPartialView`
- New partial templates: `odds_board_body.html`, `odds_table_body.html`

### Notifications Container
- `base.html` now includes a fixed-position notifications container (`#notifications`) for future bet settlement toasts (Phase 5)
