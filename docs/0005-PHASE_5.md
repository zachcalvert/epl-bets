# Phase 5: Betting Flow

## Status: Complete ✅

**Depends on:** Phase 4 (Real-Time Features) — complete
**Ref:** See `0000-PLAN.md` §Implementation Order, items 22–25

## Overview

Phase 5 implements the complete betting lifecycle: user authentication, bet placement with form validation, balance management with atomic transactions, bet settlement via Celery tasks, and a My Bets page with full history and P&L tracking.

## What Was Built

### Authentication System
- **`website/forms.py`** — `SignupForm` (email + password with confirmation) and `LoginForm` with validation
- **`website/views.py`** — `SignupView`, `LoginView`, `LogoutView` with redirect handling
- **`website/templates/website/login.html`** — Login page with error display
- **`website/templates/website/signup.html`** — Signup page with field-level error display
- **`website/urls.py`** — Auth routes: `/login/`, `/signup/`, `/logout/`
- Auto-creates `UserBalance` (1,000 credits) on signup

### Bet Placement
- **`betting/forms.py`** — `PlaceBetForm` with decimal validation (min 0.50, max 500.00, step 0.50)
- **`betting/views.py`** — `PlaceBetView` handles both full form and quick bet submissions:
  - Validates match is SCHEDULED or TIMED
  - Looks up best odds for the selected market
  - Atomic transaction: deducts balance and creates BetSlip in one operation
  - Insufficient balance check with formatted error message
  - Returns HTMX partials (bet form with errors, or confirmation)
- **`betting/templates/betting/partials/bet_form.html`** — Full bet form with radio selection buttons, stake input, and error display
- **`betting/templates/betting/partials/quick_bet_form.html`** — Inline quick bet form loaded from odds board
- **`betting/templates/betting/partials/bet_confirmation.html`** — Success state after bet placement

### Bet Settlement
- **`betting/tasks.py`** — `settle_bets` Celery task:
  - Triggered when a match status changes to FINISHED
  - Determines winning selection based on final score (home win / draw / away win)
  - Updates all pending BetSlips: marks as WON or LOST
  - Calculates and sets payout for winning bets (stake × odds)
  - Credits winnings to user balances atomically

### My Bets Page
- **`betting/views.py`** — `MyBetsView` (login required):
  - Queries user's bet history ordered by most recent
  - Calculates aggregate stats: current balance, total staked, total payout, net P&L
- **`betting/templates/betting/my_bets.html`** — Full page with:
  - 4-stat card grid (Balance, Total Staked, Total Won, Net P&L)
  - Bet history table with match, pick, odds, stake, payout, status, and date
  - Color-coded status badges (Pending=amber, Won=green, Lost=red, Void=gray)
  - Color-coded P&L (green for positive, red for negative)
  - Empty state with CTA to odds board

### Quick Bet from Odds Board
- **`betting/views.py`** — `QuickBetFormView` returns the inline quick bet form partial
- Odds buttons on the odds board use `hx-get` to load the form inline below the match row
- Form submits via `hx-post` and swaps the confirmation partial in place

### Navbar Updates
- Balance display for authenticated users (desktop + mobile)
- "My Bets" nav link (authenticated only) with active state highlighting
- Auth buttons: Login/Signup for anonymous, balance + Logout for authenticated

### Configuration
- **`config/settings.py`** — Added `LOGIN_URL` setting
- **`config/urls.py`** — Added website URLs at root path
