# Phase 8: CI/CD

## Status: In Progress

**Depends on:** Phase 7 (Community & Testing) — in progress
**Ref:** See `LAUNCH-PLAN.md` §Phase 8

## Goal

Add a repeatable CI workflow that enforces the repository’s quality bar on every push to `main` and every pull request.

## Proposed Scope

### 8A — GitHub Actions Workflow

- Create `.github/workflows/ci.yml`
- Trigger on pushes to `main` and pull requests
- Run `ruff check .`
- Run pytest with coverage
- Fail the workflow when the configured coverage gate is missed

## Implemented Slice

The first implementation pass adds `.github/workflows/ci.yml` with:

- `push` to `main` and `pull_request` triggers
- native Python setup via `actions/setup-python`
- Poetry dependency installation
- PostgreSQL and Redis service containers
- `ruff check .`
- pytest coverage run for `core`, `users`, and `website`
- conditional Codecov upload when `CODECOV_TOKEN` is configured

### 8B — Runtime Strategy

Choose one of these approaches before implementation:

1. Native Poetry install in Actions
   - Faster feedback
   - Less environment parity with local Docker usage
2. Docker-based CI
   - Higher parity with local commands
   - Slower and more operationally heavy

### 8C — Services and Secrets

- Confirm whether PostgreSQL and Redis should be provisioned as GitHub Actions services
- Decide whether API keys are needed in CI or whether tests should remain fully mocked and self-contained
- Decide whether Codecov upload should happen in this phase or immediately after the public repo step in Phase 9

## Recommended Starting Point

Use native Poetry install plus GitHub Actions service containers for PostgreSQL and Redis. That keeps CI simpler than full Docker-in-Docker while still exercising the Django app against realistic backing services.

## Open Decisions

- Keep CI scoped to `core`, `users`, and `website` initially, or expand tests before wiring the workflow
- Upload coverage to Codecov now, or defer until the public GitHub repo is live
- Add a README badge in this phase, or roll it into the Phase 9 publish pass
