# Phase 8: CI/CD

## Status: Complete

**Depends on:** Phase 7 (Community & Testing) — in progress
**Ref:** See `LAUNCH-PLAN.md` §Phase 8

## Goal

Add a repeatable CI workflow that enforces the repository’s quality bar on every push to `main` and every pull request.

## Proposed Scope

### 8A — GitHub Actions Workflow

- Create `.github/workflows/ci.yml`
- Trigger on pushes to `main` and pull requests
- Run `ruff check .`
- Run pytest with coverage
- Fail the workflow when the configured coverage gate is missed

## Implemented Slice

Phase 8 now ships `.github/workflows/ci.yml` with:

- `push` to `main` and `pull_request` triggers
- native Python setup via `actions/setup-python`
- Poetry dependency installation
- PostgreSQL and Redis service containers
- `ruff check .`
- full pytest suite with repo-wide coverage enforcement from `pyproject.toml`
- conditional Codecov upload when `CODECOV_TOKEN` is configured
- Python 3.11 to match the project runtime target
- CI and Codecov badges in `README.md`

### 8B — Runtime Strategy

Implemented: native Poetry install in Actions with PostgreSQL and Redis service containers. This keeps feedback faster than Docker-in-Docker while still exercising the Django app against its real backing services.

### 8C — Services and Secrets

- PostgreSQL and Redis are provisioned as GitHub Actions services
- CI uses placeholder API-key env vars because tests stay mocked and self-contained
- Codecov upload is enabled conditionally when `CODECOV_TOKEN` is configured

## Outcome

Phase 8 now enforces the repo's current quality bar in CI instead of the earlier partial slice. The workflow runs the same full covered app set tracked by coverage locally, and the README exposes CI and Codecov status for the eventual public repo.
